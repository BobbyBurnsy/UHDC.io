Architectural Philosophy

The platform aligns operational efficiency with strict data sovereignty. The framework contains zero hidden telemetry, ensuring all organizational execution data, asset maps, and audit logs remain exclusively within your tenant’s controlled environment.

Identity-First RBAC

The UHDC eliminates secondary credential silos. It utilizes Pass-Through Authentication via the Connect-MgGraph module, operating strictly on Delegated Permissions. It cannot grant technicians any access they do not already natively possess.

Dynamic Domain Filtration

To mitigate cross-tenant data leakage in multi-agency environments, the engine programmatically extracts the technician’s UPN. It mathematically restricts all AD and Graph searches to their specific organizational unit.

Immutable Audit Trails

Every execution—from a Browser Reset to a Cloud LAPS extraction—is piped into a centralized, timestamped CSV database via SMB (Port 445). It programmatically stamps the technician’s identity and the target hardware ID for forensic review.

Access Hierarchy & Authorization

The UHDC does not use overarching application permissions or hidden service principals. Graph API requests are only authorized if the technician's account is already assigned the corresponding roles within the Microsoft 365/Intune portal.

Operational Safety & Concurrency

How does the UHDC handle the PowerShell "Double-Hop" problem securely?
Operational governance is achieved through strict session isolation. Tools like "Remote Install" utilize PsExec under the NT AUTHORITY\SYSTEM context. By executing as SYSTEM, the target machine authenticates to the network as its computer account (DOMAIN\COMPUTER$), allowing it to pull installers from shares that standard WinRM sessions cannot reach, while keeping the execution completely hidden from the user in Session 0.
How is data integrity maintained during massive network sweeps?
To ensure data integrity, the GlobalNetworkMap engine employs an Atomic Save operation. The script executes a save every 50 loops, writing to a temporary .tmp file and performing a byte-length health check before instantly swapping it with the live JSON database. This prevents database corruption during network drops and ensures a .bak duplicate is always available for recovery.
Does the console lock up during long executions?
No. The UHDC utilizes Runspace Pools. This multi-threaded architecture ensures the graphical interface remains responsive while background threads handle intensive tasks like network scans or remote deployments. This ensures the technician remains in control of the console at all times, raising the floor of IT capability while significantly lowering the organizational risk profile.

Bypass the Event Viewer GUI

When an application silently crashes or a laptop blue-screens, standard protocol requires remote-controlling the machine and manually digging through the slow Windows Event Viewer. This engine extracts the exact faulting module in seconds.

Resilient Execution Pipeline

Establishes a remote session over Port 5985 (WinRM) to execute the query locally on the target. If endpoint firewalls block WinRM, the engine seamlessly falls back to a Base64-encoded payload deployed via PsExec.

Dynamic UI Injection

Packages the raw event objects into a compressed JSON string, transmits it back to the console, and pipes it into a dynamic HTML <table> for immediate, color-coded visual triage.

Silent CSV Backup

Simultaneously serializes the full, un-truncated dataset to a local, timestamped CSV file in your C:\UHDC\Logs directory to ensure deep forensic review is available if the UI snippet isn't enough.

Technical Q&A

Q: What network ports are required for this tool to function?
The engine primarily relies on Port 5985 (WinRM/HTTP) to execute the query locally on the target machine, which is significantly faster and more reliable than pulling raw logs across the network. If WinRM is blocked, it falls back to Port 445 (SMB) and Port 135 (RPC) to deploy the PsExec payload.
Q: Why does the deep scan limit the search to the last 10,000 events?
Windows Event logs can easily contain millions of entries. Attempting to parse the entire log can cause severe pipeline lag, memory spikes, and UI lockups. A 10,000-event buffer is the perfect architectural balance—it safely captures recent forensic data while maintaining a sub-10-second response time for the technician.
Q: Why does the dashboard table only show 50 logs, but the CSV has all of them?
Browser DOM performance is highly sensitive to massive HTML tables. Flooding the web UI with thousands of multiline error strings would cause the interface to freeze and ruin the user experience. We limit the "Heads Up Display" to the top 50 results for immediate triage, and export the full, un-truncated dataset to CSV for deep forensic review.
Q: Does this query the remote 'Security' log too?
No, it specifically targets the 'System' and 'Application' logs. The 'Security' log is heavily restricted, often requiring elevated Domain Admin auditing privileges to read remotely. Furthermore, it is typically flooded with thousands of routine logon/logoff audits, which dramatically slows down query times for standard Tier 1/Tier 2 troubleshooting.
